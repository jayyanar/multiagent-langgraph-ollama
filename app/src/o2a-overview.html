Problem

SOPs don't scale through humans

Operations teams repeat the same multi-source validations daily. Agents can do it — but only if they know the data. Today they don't.

Solution

Two governed platform services

DDaaS catalogs every source and makes agents data-aware. Tool Registry executes their calls through pooled, audited connections.

Outcome

Each new SOP gets cheaper

Onboard a source once; every future agent inherits it. Marginal cost of the next automation drops sharply.

How to read this diagram

  • SKAN.ai captures what operations teams do.
  • Ingestion Pipeline turns that into a procedural SKILL.md.
  • DDaaS tells the pipeline what data exists, what it means, and where to find it, so the SKILL.md is bound to real Tools at build time.
  • Tool Registry is where the live agent actually calls data sources — through pooled, audited connections.
  • Google ADK orchestrates the agent run; ADK never touches a database driver.
  • Enterprise data sources are federated, not replaced — DBs, APIs, S3 stay where they are.

Why this works at scale

  • One catalog, many consumers — every team's agents get data-awareness from the same source of truth.
  • One execution choke point — every read in the company funnels through Tool Registry, so quotas, timeouts, audits, and circuit breakers apply uniformly.
  • One credential store — Vault is the only place a password lives.
  • Two services, two scale signals — DDaaS scales on catalog reads, Tool Registry scales on active queries. Each grows where it needs to.
  • Compounding leverage — onboarding cost is paid once per source, not once per agent.
